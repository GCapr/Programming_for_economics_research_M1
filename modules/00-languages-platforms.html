0.1 Comparing Python, Stata, and R

Before diving into interfaces, let me explain what makes each language unique. These tools emerged from different communities with different goals, and understanding their origins helps you choose the right one for each task.

Aspect Python Stata R
Born 1991 1985 1993
Origin General-purpose scripting Statistical analysis Statistical computing
License Free, open-source Commercial (paid) Free, open-source
Strength ML, versatility, industry Econometrics, panel data Statistics, visualization
Learning curve Gentle Gentle for basics Steeper initially

In my experience, most researchers eventually become proficient in at least two of these languages. Economics tends to favor Stata for publication, but Python and R are increasingly common, especially for machine learning applications.

Stata Interface — The Do-file Editor (Window > Do-file Editor) is where you write scripts. Always work in .do files for reproducibility!
Stata Workflow Tip

I cannot stress this enough: never work only in the Command window. Always use the Do-file Editor (Ctrl+9) to write scripts. This ensures your work is reproducible. The Command window is for quick exploration; serious analysis goes in .do files.

0.5 Google Colab

Google Colab (Colaboratory) is a free, cloud-based Jupyter notebook environment. You don't need to install anything—it runs entirely in your browser with free GPU access. I often use it for quick experiments and when teaching.

Why Use Google Colab?
  • Zero setup: Python environment ready instantly
  • Free GPU/TPU: Train ML models without expensive hardware
  • Easy sharing: Share notebooks like Google Docs
  • Google Drive integration: Mount your Drive to access files
  • Collaboration: Multiple people can edit simultaneously

0.6 AI-Assisted Coding Tools

AI coding assistants can dramatically speed up your workflow. They write boilerplate code, explain errors, and help translate between languages. I encourage you to experiment with these tools—they're becoming essential for modern research.

Claude Code

Claude Code is Anthropic's command-line tool for AI-assisted coding. Unlike autocomplete tools, you have conversations with Claude about your code. 

# Install Claude Code
$ npm install -g @anthropic-ai/claude-code

# Authenticate
$ claude auth

# Start Claude in your project
$ cd my-research-project
$ claude

# Example interactions:
> "Read my analysis.py and explain what it does"
> "Add clustered standard errors to the regression"
> "Convert this Stata code to Python"
> "Why am I getting this error? [paste error]"
> "Write a function to bootstrap confidence intervals"

# Claude can read, edit, and create files in your project

GitHub Copilot

GitHub Copilot provides real-time code suggestions as you type. It's trained on billions of lines of code and can complete entire functions from comments.

Copilot Pricing

$10/month for individuals, but free for students and educators. Apply at education.github.com with your .edu email.

LM Studio: Run Local Models

LM Studio lets you run large language models locally on your computer. This is useful for:

  • Privacy: Code never leaves your machine
  • Offline work: No internet required
  • Cost: Free after download
  • Experimentation: Try different open-source models

Download from lmstudio.ai. Requires a computer with at least 8GB RAM (16GB+ recommended).

0.7 Choosing the Right Tool

Here's my practical guide for choosing the right tool:

If you need to... Use...
Publish in economics journals Stata (standard) or R
Do machine learning Python (only real option)
Create beautiful visualizations R (ggplot2 is unmatched)
Work with complex survey data Stata (svy commands are excellent)
Work in industry/tech Python (lingua franca)
Quick exploration, no install Google Colab
One editor for everything VS Code
My Recommendation

Learn all three at a basic level, then specialize based on your field and collaborators. This course gives you enough of each to read others' code and translate between languages. By the end, you'll be able to choose the right tool for each task.
