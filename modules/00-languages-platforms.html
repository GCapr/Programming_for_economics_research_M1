0.1 Three Languages, Three Philosophies

Before diving into code, it's essential to understand that Python, Stata, and R aren't just different syntaxes‚Äîthey represent fundamentally different approaches to data analysis, born from different communities with different needs.

Python: The General-Purpose Powerhouse

Python was created in 1991 by Guido van Rossum as a general-purpose programming language. Its philosophy emphasizes code readability and simplicity ("There should be one‚Äîand preferably only one‚Äîobvious way to do it"). Python wasn't designed for statistics; it became a data science tool through community-built libraries like NumPy (2006), pandas (2008), and scikit-learn (2010).

üêç Python's Zen

Type import this in Python to see "The Zen of Python"‚Äî19 aphorisms that capture the language's philosophy: "Beautiful is better than ugly. Explicit is better than implicit. Simple is better than complex."

Key characteristics:

  • Object-oriented: everything is an object with methods
  • Zero-indexed: arrays start at position 0
  • Indentation matters: whitespace defines code blocks
  • Extensive ecosystem: web development, machine learning, automation
  • Free and open-source

Stata: The Econometrician's Companion

Stata was created in 1985 by William Gould specifically for statistical analysis. Unlike Python or R, Stata is a commercial product developed by StataCorp, with a focus on reliability, documentation, and reproducibility. The Stata community is centered in economics, political science, and epidemiology.

Key characteristics:

  • One dataset in memory: Stata works on a single rectangular dataset at a time
  • Command-based: each line is typically one command
  • Excellent documentation: every command has detailed help files
  • Built-in econometrics: panel data, IV, survival analysis out of the box
  • Proprietary: requires a license (~$125-595 for students)

R: The Statistician's Workshop

R was created in 1993 by Ross Ihaka and Robert Gentleman at the University of Auckland as an open-source implementation of the S language (developed at Bell Labs in the 1970s). R was designed by statisticians, for statisticians, which shows in its design choices.

Key characteristics:

  • Vector-oriented: operations naturally apply to entire vectors
  • One-indexed: arrays start at position 1 (like humans count)
  • Functional programming: emphasizes functions over objects
  • Incredible package ecosystem: CRAN hosts 20,000+ packages
  • Free and open-source (GNU GPL)

0.2 Detailed Comparison

Syntax Philosophy

Let's see how the same operation‚Äîcalculating the mean of a variable by group‚Äîlooks in each language:

# Python: Object-oriented, method chaining
import pandas as pd

# Load data
df = pd.read_csv("data.csv")

# Calculate mean by group
result = df.groupby('continent')['lifeExp'].mean()

# Note: The DataFrame 'df' is an object
# .groupby() and .mean() are methods of that object
# This is method chaining: object.method1().method2()
* Stata: Command-based, operates on data in memory

* Load data (replaces whatever was in memory)
use "data.dta", clear

* Calculate mean by group
collapse (mean) lifeExp, by(continent)

* Note: Each line is a command
* 'collapse' transforms the dataset in memory
* There's no assignment operator needed

* Alternative: tabstat for display only
tabstat lifeExp, by(continent) stat(mean)
# R: Functional + tidyverse pipe syntax
library(tidyverse)

# Load data
df <- read_csv("data.csv")

# Calculate mean by group (tidyverse style)
result <- df %>%
  group_by(continent) %>%
  summarize(mean_life = mean(lifeExp))

# Note: The pipe %>% passes output to next function
# Read as: "take df, THEN group by, THEN summarize"
# <- is the assignment operator (= also works)

Indexing: Zero vs One

One of the most common sources of confusion when switching between languages:

# Python: ZERO-INDEXED
my_list = ['a', 'b', 'c', 'd', 'e']

my_list[0]      # 'a' - first element
my_list[1]      # 'b' - second element
my_list[-1]     # 'e' - last element
my_list[0:3]    # ['a', 'b', 'c'] - elements 0, 1, 2 (not 3!)

# DataFrames
df.iloc[0]      # First row
df.iloc[0, 0]   # First row, first column
* Stata: ONE-INDEXED

* Access observation (row) 1
display var1[1]     // First observation

* In brackets notation
list in 1           // First row
list in 1/3         // Rows 1, 2, and 3

* _n gives current observation number
gen row_num = _n    // 1, 2, 3, ...
# R: ONE-INDEXED
my_vector <- c('a', 'b', 'c', 'd', 'e')

my_vector[1]        # 'a' - first element
my_vector[2]        # 'b' - second element
my_vector[1:3]      # c('a', 'b', 'c') - elements 1, 2, 3 (inclusive!)

# DataFrames
df[1, ]             # First row
df[1, 1]            # First row, first column

0.3 Choosing the Right Tool

üí° The Best Language Is...

...the one your colleagues and field use. Collaboration, existing codebases, and institutional support often matter more than technical features.

Choose Python if:

  • You're in tech, data science, or computer science
  • You need machine learning or deep learning
  • You want to build applications, not just analyze data
  • You're scraping web data or working with APIs
  • You want maximum career flexibility

Choose Stata if:

  • You're in economics, political science, or epidemiology
  • Your advisors and coauthors use Stata
  • You value excellent documentation and support
  • You want built-in, tested econometric methods
  • You prioritize reproducibility and stability

Choose R if:

  • You're in statistics, biostatistics, or social sciences
  • You need cutting-edge statistical methods
  • You want the best visualization tools (ggplot2)
  • You're doing spatial analysis or bioinformatics
  • You prefer open-source and community development

0.4 Development Environments (IDEs)

An Integrated Development Environment (IDE) is where you write, run, and debug code. Choosing the right IDE can dramatically improve your productivity.

RStudio (for R)

RStudio is the de facto standard IDE for R, developed by Posit (formerly RStudio, PBC). It's specifically designed for data science workflows.

Key features:

  • Four-pane layout: Source, Console, Environment, Files/Plots/Help
  • R Markdown integration for literate programming
  • Git integration built-in
  • Package management and project system
  • Free Desktop version; paid Server and Cloud versions

Download: posit.co/download/rstudio-desktop

Stata's Built-in Interface

Stata comes with its own integrated interface featuring a command window, results viewer, variables manager, and do-file editor. Most Stata users work exclusively within this environment.

Key features:

  • Do-file editor with syntax highlighting
  • Variable and data browser
  • Command history and review window
  • Integrated help system

JupyterLab (for Python)

Jupyter notebooks allow you to mix code, output, and narrative text in a single document. They're excellent for exploratory analysis and teaching.

Key features:

  • Cell-based execution (run code in chunks)
  • Rich output: tables, plots, even interactive widgets
  • Export to HTML, PDF, slides
  • Supports Python, R, Julia, and 40+ languages

0.5 Visual Studio Code

Visual Studio Code (VS Code) is a free, open-source code editor from Microsoft that has become the most popular IDE in the world. Unlike language-specific IDEs, VS Code works with virtually any programming language through extensions.

üìä VS Code Adoption

According to the 2023 Stack Overflow Developer Survey, VS Code is used by 73% of developers, making it by far the most popular development environment.

Why VS Code?

  • One tool for everything: Python, R, Stata, JavaScript, LaTeX, Markdown...
  • Incredible extension ecosystem: 50,000+ extensions
  • Built-in Git: Version control without leaving the editor
  • Integrated terminal: Run commands without switching windows
  • Remote development: Work on servers, containers, or WSL seamlessly
  • AI integration: GitHub Copilot, Continue, and other AI assistants

0.6 AI-Powered Coding Tools

The emergence of large language models has revolutionized how we write code. AI assistants can help you write, debug, and explain code‚Äîdramatically accelerating your learning and productivity.

Claude Code

Claude Code is Anthropic's command-line tool that brings Claude's capabilities directly into your terminal. Unlike chat interfaces, Claude Code can read your files, execute commands, and make changes to your codebase.

ü§ñ What Claude Code Can Do
  • Read and understand your entire codebase
  • Write and edit files directly
  • Run commands and interpret output
  • Debug errors by examining stack traces
  • Refactor code across multiple files
  • Explain complex code in plain language

Installing Claude Code

# Install via npm (requires Node.js)
npm install -g @anthropic-ai/claude-code

# Authenticate with your API key
claude-code auth

# Start a session in your project directory
cd my-project
claude-code

Example Usage

# In your terminal, after starting claude-code:

> Read my analysis.py file and explain what it does

> There's a bug in line 45 - the merge is creating duplicates. Fix it.

> Add robust standard errors to all the regression specifications

> Create a new function that calculates summary statistics by group

LM Studio

LM Studio lets you run large language models locally on your own computer. This is valuable for privacy-sensitive work, offline use, or when you want to experiment with different open-source models.

Key features:

  • 100% local: Your data never leaves your computer
  • No API costs: Run unlimited queries
  • Model variety: Access Llama, Mistral, CodeLlama, and more
  • Local API server: Use with other tools via OpenAI-compatible API

Setting Up LM Studio

  1. Download: lmstudio.ai
  2. Browse models: Search for models in the Discover tab
  3. Download a model: Start with something like codellama-7b for coding tasks
  4. Load and chat: Select your model and start a conversation
  5. Start local server: Enable the API server to use with VS Code extensions
‚ö†Ô∏è Hardware Requirements

Running LLMs locally requires significant RAM. A 7B parameter model needs ~8GB RAM; a 13B model needs ~16GB. Larger models (70B) require high-end GPUs.

GitHub Copilot

GitHub Copilot integrates directly into VS Code and provides real-time code suggestions as you type. It's trained on public code and can autocomplete entire functions.

Cost: $10/month (free for students and open-source maintainers)

0.7 Building Your Workflow

The best workflow combines the right tools for each task. Here's a suggested setup for a data science researcher:

Recommended Setup

  1. Primary IDE: VS Code with language extensions
    • Use for Python, R scripts, and editing any code files
    • Integrated terminal for running commands
    • Git integration for version control
  2. Stata: Use Stata's native interface
    • Better integration with Stata's help system
    • Data browser works seamlessly
  3. Notebooks: Jupyter (Python) or Quarto (R/Python)
    • For exploratory analysis
    • For creating reports and presentations
  4. AI Assistant: Claude (chat or code) + Copilot
    • Claude for complex reasoning and explanations
    • Copilot for quick autocomplete
